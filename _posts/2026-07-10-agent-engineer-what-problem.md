从2022年底的ChatGPT横空出世，chat Bot已经逐渐占据了人们的视野，好像AI=聊天机器人。但聊天从来不是终点。真正有用的 AI，得能替你把事办完，而不只是把话说完。

让chat Bot写封邮件，它写得又快又好，但写完也就结束了——剩下的事得人自己来：复制、粘贴、打开邮箱、填收件人、发送。它能把话说漂亮，但不动手。

Agent 接管的就是这后半截。给它一个目标——把这封邮件发给张三——它会自己拆成几步，该调什么工具调什么工具，一步步自己决策怎么做，然后完成任务。

![chatbot 只会说，Agent 会动手](/assets/img/agent/chatbot_vs_agent_comic.png)

从"会说"到"会做"，听起来只多了一步，难度却完全不是一个量级。因为一旦让模型自己动手、自己决策，问题就接踵而至了。

# 难点：一个确定的世界，撞上一个不确定的大脑

传统软件工程的思维，是建立在"确定性"之上的；1 + 1 永远是 2，同样的入参永远给出同样的结果。这种确定性是过去几十年软件工程的地基——调试习惯、测试方法都是建立在此之上。

然而大模型不是这么工作的，它是个概率模型，输出是从一个概率分布里挑出来的，所以会飘、会自由发挥，还会在一件需要很多步的事情里，把一点小偏差慢慢放大成大错。

> 这个"不确定"是哪来的？现在的大模型骨子里是 **Transformer**。抛开数学不谈，它干的事其实就一件：**根据前面给的所有词，算出"下一个词是什么"**。而且算出来的不是一个确定的词，是词表里每个词各自的概率。真正的关键是最后一步——<mark><strong>它不总挑概率最高的那个，而是按这些概率来选</strong></mark>。就是这一下，让同样的开头能接出不同的下文。

![Transformer 预测下一个词：按概率选](/assets/img/agent/transformer_next_token_comic.png)

所以那个不确定，不是哪里出了错，它是这套机制本来就该有的样子。连着选几百上千个词，再放进一个要调好几十次模型的 Agent 里，那点小随机就攒成了"今天和昨天不一样"。

矛盾就在这儿：**用一个天生带着随机性的东西，去搭一个每次都得靠谱交付的系统**，这两件事本来就天然冲突。

- 传统软件像一个只会照章办事的员工，交代怎么做就怎么做，稳，但也就这样了。
- Agent 更像一个很聪明、但偶尔走神的实习生，能应付你没预想到的情况，可你也不敢把他一个人扔那儿就等着收活——得给他配好工具，让他有地方记东西，教他碰上复杂的事怎么拆，最后还得有本事检查他干得对不对。

![照章办事的员工 vs 会走神的实习生](/assets/img/agent/train_vs_intern_comic.png)

大脑天生不够靠谱，但又非用不可，那"怎么让它靠谱"这件事本身，就成了一门单独的学问。它跟传统软件工程不一样，也跟算法建模不一样。

- 传统软件工程默认每一步都是确定的；
- 算法团队管的是大脑本身够不够聪明；
- 而 Agent 工程正好夹在中间，专门管一件他们俩都不管的事：**怎么在不确定的基础上，做出确定来**

因此，Agent开发总的说来有五件事要解决：

1. **该不该放手**。什么时候该让模型照着固定流程走，什么时候又该松手让它自己找路，这个分寸不好拿捏。
2. **怎么给它好用的工具**。大脑再聪明，工具难用也白搭。给 Agent 设计工具，跟给人设计一个好用的界面，其实是一回事。
3. **怎么让它记得住**。这大脑有个尴尬的地方，基本是"一上班就失忆"的，那它凭什么记得三步之前发生过什么。
4. **怎么让它走得远又不走丢**。答一句话容易，连着走十步不出岔子难，得想办法把一次放手切成一步步都还看得住。

说到底，Agent 现在还远没到"靠谱"的地步，这五件事，哪件都没有现成答案。模型越来越强，它们会不会有一天自己就消化掉了？也可能反过来，越强的模型越难管，谁也说不准。

但至少现在，这些麻烦事，还得靠人一件件去料理。
